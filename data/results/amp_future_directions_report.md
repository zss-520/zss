## 1. 评测结果总体结论

基于当前可用的 `AUROC` 与 `AUPRC` 数据，结合会议决议的动态指标权重体系，得出以下结论：

1. **整体表现强弱**：在仅依赖排序指标的代理排名中，`AMPlify`（加权秩分 0.729）与 `Macrel`（0.708）表现相对领先，`amPEPpy`（0.375）居中，`AMP-Scanner-v2`（0.083）垫底。该排名严格基于可用指标的秩次加权，**不代表综合性能**。
2. **跨数据集稳定性**：所有参评模型均未表现出稳定的跨数据集泛化能力。`Macrel` 在 `AntiBP2_数据集` 和 `Dataset_1_test1` 上 AUROC 突破 0.90，但在 `AIP_数据集` 骤降至 0.54；`AMPlify` 同样在 `AIP_数据集` 出现性能断崖。这表明模型高度依赖特定数据集的分布特征，缺乏真正的跨域鲁棒性。
3. **数据集敏感性**：`AIP_数据集` 充当了严格的分布外（OOD）测试集，所有模型在此处的 AUROC 均逼近随机猜测（0.50~0.57），暴露出严重的 dataset-specific overfitting。
4. **证据局限性声明**：**当前结果不足以支持关于模型“真实筛选价值”或“综合性能强弱”的确定性结论。** 由于会议决议赋予最高权重的 `MCC (0.35)` 与次高权重的 `Recall (0.15)` 完全缺失，任何基于现有数据的性能推断仅限于“全局概率排序能力”，无法验证模型在固定工作阈值下的漏报控制与误报代价。强行 extrapolate 将违背学术审计原则。

## 2. 跨数据集泛化能力分析

1. **波动性量化**：根据 `performance_summary`，各模型 AUROC 标准差普遍较高（Macrel: 0.154, AMPlify: 0.139, AMP-Scanner-v2: 0.129, amPEPpy: 0.102）。高方差直接反映模型对数据集构建协议（如负样本生成方式、长度截断策略、物种来源）的高度敏感。
2. **Dataset-Specific Overfitting 判断**：`Macrel` 与 `AMPlify` 在历史平衡集（AntiBP2, Dataset_1）上的优异表现，极可能源于对短序列片段、特定电荷区间或低复杂度背景的过拟合。当面对 `AIP_数据集` 这种经过严格理化性质匹配（长度±10aa、电荷±1、疏水性对齐）的困难负样本时，模型无法区分正负类，证明其并未学习到抗菌肽的真实生物物理作用机制（如两亲性螺旋、膜穿透势垒）。
3. **Hard Negative 与分布迁移场景**：当前证据显示，所有模型在分布迁移场景下性能均发生显著退化。`AMP-Scanner-v2` 在 `sAMPpred-GAT_测试集` 上 AUROC 仅为 0.625，进一步印证浅层 CNN 架构难以捕获跨谱系的保守特征。**当前证据不足以证明任何参评模型具备可靠的跨数据库泛化能力。** 若需验证，必须在严格去冗余（CD-HIT 80%/Taxon划分）的外部独立测试集上进行盲测。

## 3. 动态指标维度分析

严格围绕会议决议指标体系逐项剖析：

| 指标 | 权重 | 衡量内容 | 对AMP真实筛选的意义 | 当前表现与暴露问题 |
|:---|:---|:---|:---|:---|
| **ACC** | 0.1000 | 整体分类正确率 | 在极度不平衡场景下极易产生“多数类幻觉”，参考价值有限。 | **缺失 (NaN)**。无法评估模型在正负样本比例偏移时的基础判别底线。 |
| **Recall** | 0.1500 | 正类检出率（灵敏度） | 对应“初筛宁可误报不可漏报”的工程红线。低 Recall 意味着保守家族成员被系统性漏检。 | **缺失 (NaN)**。无法验证模型是否会为了刷高 Precision 而牺牲灵敏度，湿实验漏报风险未知。 |
| **MCC** | 0.3500 | 混淆矩阵四象限无偏度量 | 抗类别偏移的全局稳健性指标。同时惩罚 FP 与 FN，是衡量模型泛化质量的黄金标准。 | **缺失 (NaN)**。权重最高的指标缺失导致加权总分失效。无法确认高 AUPRC 是否以特异性崩塌为代价。 |
| **AUROC** | 0.1000 | 全局正负类排序能力 | 反映模型对正类优先级的宏观把握，但对极端不平衡下的正类富集细节不敏感。 | **可用**。均值 0.65~0.81，但波动极大。低权重符合其在失衡场景下的局限性，仅作为辅助参考。 |
| **AUPRC** | 0.3000 | 正类 Precision-Recall 轨迹面积 | 极端不平衡下的黄金标准，直接映射“从海量非活性背景中淘出活性肽”的初筛富集效率。 | **可用**。均值 0.68~0.83，是当前的核心代理指标。但 AUPRC 高仅证明排序优秀，不保证在实用阈值下的高 Precision。 |

**综合结论**：当前评测体系存在结构性断裂。0.60 的权重（ACC+Recall+MCC）因流水线兼容性危机未能产出，导致评估退化为纯排序能力测试。科学上，**AUPRC 仅提供了“门票资格”，而 Recall 与 MCC 才是决定“转化率与资源消耗”的关键**。缺失项使得我们无法回答“模型在实际高通量筛选中每验证100条候选肽能稳定捕获多少活性肽”这一核心工程问题。

## 4. 现有 AMP 预测模型的主要局限

1. **数据泄漏与同源污染风险**：历史模型多在早期数据库（DRAMP v1/v2, AntiBP2）上训练，这些数据集内部存在较高的序列相似性。模型很可能通过记忆同源序列而非学习 motif 获得高分，在严格去冗余测试集上必然遭遇性能坍塌。
2. **负样本过于简单**：多数传统模型使用随机基因组片段或无关蛋白作为负例。真实筛选场景中，负样本多为具有类似理化性质但无活性的结构化蛋白。模型缺乏对“困难负例”的判别训练，导致泛化边界模糊。
3. **平衡数据集与真实极端不平衡场景的差距**：Benchmark 常采用 1:1 或轻度失衡设置，而实际筛选比例为 1:50~1:100。模型在平衡集上优化的 BCE 损失函数无法直接映射到真实管线的 Top-K 富集需求。
4. **过度依赖浅层氨基酸组成特征**：Macrel 与 amPEPpy 依赖 PseAAC、Dipeptide Composition 等手工特征，无法捕获长程静电相互作用、疏水斑块分布及构象倾向，架构容量已触及天花板。
5. **缺乏跨数据库泛化能力**：如前所述，跨分布测试性能骤降表明模型未解耦序列特征与数据集偏差，不具备真正的生物学普适性。
6. **可复现性与工程部署障碍**：Legacy 模型多直接输出 Logits 或未校准概率，缺乏标准化的决策阈值（Decision Threshold）与置信度区间。与新评测 Harness 的阈值冻结协议不兼容，直接导致核心指标 NaN，阻碍工业级落地。

## 5. 下一代 AMP 预测模型的发展建议

结合 benchmark 结果与指标缺失情况，提出以下可执行方向：

1. **Hard Negative Learning 策略**：针对 `AIP_数据集` 性能崩溃现象，下一代模型必须引入理化性质严格匹配的困难负样本进行对抗训练或难例挖掘（Hard Example Mining），迫使模型学习真实的膜相互作用特征而非数据集捷径。
2. **Protein Language Model 与物化特征融合**：摒弃纯手工特征，采用 ESM-2/ProtBERT 等预训练嵌入作为基础表征，并与显式的电荷、疏水性、两亲性指数张量进行跨模态融合。这可直接解决浅层特征表达力不足的问题，提升跨谱系泛化。
3. **阈值校准与概率标准化管线**：强制要求所有模型输出经过 Platt Scaling 或 Isotonic Regression 校准的概率分布，并在验证集上通过 Youden Index 锁定最优阈值后，在测试集冻结复现。这是补齐 `MCC` 与 `Recall` 缺失、打通加权评分体系的必要前提。
4. **筛选导向的损失函数设计**：放弃标准 BCE，采用 Focal Loss、Top-K Precision 优化或代价敏感学习（Cost-Sensitive Learning），使模型直接对齐“控制 FP 绝对数量”与“保障 Recall 底线”的业务目标，契合会议权重中 AUPRC 与 Recall 的设计意图。
5. **可解释性 Motif 分析与注意力可视化**：利用 Attention Rollout 或 Integrated Gradients 提取模型关注的功能区域，验证是否与已知 AMP 保守基序（如阳离子头-疏水尾）一致。避免黑盒模型产生统计学伪相关。
6. **Wet-Lab 闭环验证机制**：建立“干预测 → 概率排序 → 化学合成 → 体外抑菌/溶血测试”的反馈回路。将湿实验验证的 Top-50 命中率作为最终金标准，反哺模型迭代，打破纯计算指标的自嗨循环。

## 6. 下一代 AMP benchmark 应该如何建设

1. **数据集构建建议**：严格执行 `CD-HIT 90%/80%` 去冗余与 `Taxon` 跨物种划分。正样本限定为实验验证 AMP，负样本必须来自 `UniProtKB-TrEMBL` 中明确注释为 `Non-antimicrobial` 的蛋白，并进行长度/电荷/疏水性三重匹配。彻底剔除 AIP/ACP 跨界污染条目。
2. **正负样本定义建议**：统一活性阈值（如 MIC ≤ 32 μg/mL）。负样本需排除天然防御肽、信号肽及短片段随机串扰，确保标签纯净度。
3. **同源去重与数据泄漏控制建议**：发布完整的序列相似度矩阵与划分脚本。强制报告训练集-测试集最大同源百分比。若测试集内存在 >30% 的同源序列，该结果应降级标注为“记忆效应测试”而非“泛化测试”。
4. **指标体系建议**：保留会议动态权重（ACC 0.1, Recall 0.15, MCC 0.35, AUROC 0.1, AUPRC 0.3）。新增 `Recall@Top-K` 与 `Precision@Top-K` (K=50/100/500) 模拟真实截断筛选。所有核心指标必须附带 `Bootstrap 95% CI`。
5. **Leaderboard 机制建议**：设立双轨榜单：① `Ranking Ability`（AUROC/AUPRC）；② `Screening Utility`（MCC/Recall@K/F1）。禁止仅凭单一曲线指标定级。公布完整混淆矩阵与阈值快照。
6. **复现协议建议**：提供 Docker 容器化环境、固定随机种子、标准化推理脚本。要求模型作者提交预训练权重哈希值与概率校准配置文件。无完整流水线者不予入库。

## 7. 可以写进论文 Discussion / Conclusion 的核心观点

1. 历史基线模型在严格去冗余与跨分布测试集上性能显著衰减，表明其过度依赖序列相似性捷径与浅层统计特征，尚未掌握抗菌肽跨谱系作用的通用表征。
2. 仅依赖 AUROC/AUPRC 的评估存在严重局限；缺失 MCC 与 Recall 导致无法验证模型在固定工作阈值下的漏报与误报代价，当前结果不足以支持“高排序分即高筛选价值”的推论。
3. 负样本构建策略是决定模型泛化能力的核心变量；简单随机负样本训练的模型在面对理化性质匹配的困难负例时表现崩溃，亟需引入难度感知采样机制。
4. 传统特征工程（k-mer/PseAAC）已触及性能天花板，下一代模型需引入预训练蛋白语言模型与显式物化特征融合，以捕获长程电化学协同效应。
5. 评测流水线的工程兼容性（概率校准、阈值冻结）直接决定基准的科学有效性；缺乏标准化输出协议的模型不具备工业部署潜力，应强制要求交付可复现的推理管线。
6. AMP 预测研究范式应从“二分类准确率竞赛”转向“高通量富集效率优化”，未来 benchmark 需强制纳入 Top-K 截断指标与湿实验验证闭环，以真实研发成本函数为导向。

## 8. 后续实验优先级

### P0：必须马上做
- **要做什么**：修复评测流水线，实现概率校准（Platt/Isotonic）与验证集阈值冻结，补全 `MCC`、`Recall`、`ACC` 输出。
- **为什么要做**：当前核心加权指标（合计权重 0.60）全为 NaN，加权总分无法闭合，所有排名均为代理指标，违背会议决议的统计完备性要求。
- **增强说服力**：恢复指标体系完整性，使综合得分具备统计学意义，消除“伪排名”质疑，为模型横向对比提供合规基础。

### P0：必须马上做
- **要做什么**：在 `AMP-Bench-Sim80` 与 `AMP-Bench-Taxon` 跨域集上重新跑批所有参评模型。
- **为什么要做**：现有数据仅在历史平衡集表现良好，跨家族/跨物种泛化能力未知，无法排除同源过拟合。
- **增强说服力**：验证模型是否真正学到通用 AMP 特征，提供严谨的泛化边界证据，支撑论文关于“架构代差”的核心论点。

### P1：强烈建议做
- **要做什么**：引入 Hard Negative 采样策略（长度/电荷/疏水性严格匹配的非活性肽）对主流模型进行微调或重训。
- **为什么要做**：`AIP_数据集` 性能暴跌暴露模型对困难负例区分力不足，直接制约湿实验假阳性控制。
- **增强说服力**：展示模型在真实复杂背景下的特异性提升，降低合成验证成本，增强算法的工程落地价值。

### P1：强烈建议做
- **要做什么**：开发并接入基于 pLM（如 ESM-2/ProtT5）的增量模型进行对比评测。
- **为什么要做**：传统架构已见顶，pLM 能捕捉进化保守性、二级结构倾向与隐式构象信息，代表技术演进方向。
- **增强说服力**：确立新一代 SOTA 基线，直观展示深度学习范式替代传统特征工程的必要性，提升论文前沿性。

### P2：后续扩展做
- **要做什么**：构建 `Recall@Top-K` 与 `Precision@Top-K` 模拟筛选管线指标，并开展小规模湿实验交叉验证（合成 Top-50 预测肽）。
- **为什么要做**：会议权重侧重 AUPRC，但实际筛选关注截断后的绝对命中率；干预测评分与体外活性可能存在脱节。
- **增强说服力**：使 benchmark 更贴近药物发现预算约束，提供端到端证据链，大幅提升论文的转化医学说服力与行业影响力。
