## 🧬 二、结合应用场景：权重分配是否合理？

本任务属于抗菌肽预测（Antimicrobial Peptide Prediction），是典型的生物信息学分类问题，尤其关注**不平衡数据下的稳健判别能力**。

### 当前权重设定回顾：
```json
{
  "ACC": 0.1,
  "Recall": 0.15,
  "MCC": 0.35,
  "AUROC": 0.1,
  "AUPRC": 0.3
}
```

### 科学性评估：

#### ✔️ 合理之处：
- **MCC 被赋予最高权重（0.35）** 是非常合理的：
  - MCC 综合考虑 TP/TN/FP/FN 四个因素，特别适用于类别不平衡的数据集；
  - 对于抗菌肽这类正样本稀少的任务来说，MCC 是衡量模型泛化能力和鲁棒性的核心指标。

- **AUPRC 权重次之（0.3）** 也符合实际需求：
  - 抗菌肽识别本质上是一个“少数类优先”的任务，AUPRC 更注重少数类（即阳性类）的表现；
  - 相比 AUROC，AUPRC 不受负样本数量膨胀的影响，更适合此类任务。

#### ❌ 存疑之处：
- **ACC 权重过低（0.1）**：
  - 尽管 ACC 容易被不平衡数据误导，但在工业界或临床辅助决策中仍具有直观意义；
  - 若用于部署阶段的整体性能监控，ACC 仍是重要参考之一。

- **Recall 权重偏低（0.15）**：
  - 在抗菌肽发现中，“漏检”代价极高——一个潜在有效的抗菌肽若未被检测出，可能导致药物研发失败；
  - 因此，Recall 应至少与 AUPRC 或 MCC 平行对待。

### ✅ 推荐调整建议（供未来研究参考）：
```json
{
  "ACC": 0.1,
  "Recall": 0.2,
  "MCC": 0.3,
  "AUROC": 0.1,
  "AUPRC": 0.3
}
```
> 提升 Recall 至 0.2，适度降低 MCC 到 0.3，以更全面反映模型在真实应用中的风险控制能力。

## 📘 附录：MCC 计算示意（供验证）

假设使用混淆矩阵计算 MCC：

$$
\text{MCC} = \frac{TP \times TN - FP \times FN}{\sqrt{(TP+FP)(TP+FN)(TN+FP)(TN+FN)}}
$$

例如对于 AMPlify：
- 若 TP=721, TN=8500, FP=214, FN=279，则 MCC ≈ 0.45 左右，符合预期。
